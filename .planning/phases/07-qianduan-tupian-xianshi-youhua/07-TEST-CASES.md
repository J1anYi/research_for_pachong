# Phase 7 测试用例文档：前端图片显示优化

**测试日期:** 2026-04-23
**测试环境:** http://localhost:8080

---

## 测试前置条件

1. MediaCrawler 服务已启动 (`uv run uvicorn api.main:app --port 8080`)
2. 存在测试数据（小红书笔记数据）
3. 部分笔记有本地图片，部分只有远程图片

---

## 测试用例

### TC-01: API 返回本地图片路径字段

**测试目的:** 验证 API 正确返回 local_image_url 和 remote_image_url 字段

**测试步骤:**
1. 访问 API 端点 `http://localhost:8080/api/notes?page=1&page_size=10`
2. 检查响应数据中每条笔记是否包含 `local_image_url` 字段
3. 检查响应数据中每条笔记是否包含 `remote_image_url` 字段
4. 检查 `first_image_url` 字段是否正确（优先本地，否则远程）

**预期结果:**
- ✅ 每条笔记包含 `local_image_url` 字段（可为 null）
- ✅ 每条笔记包含 `remote_image_url` 字段（可为 null）
- ✅ `first_image_url` 返回有效的图片 URL

---

### TC-02: 前端优先使用本地图片

**测试目的:** 验证前端页面优先显示本地图片

**测试步骤:**
1. 打开数据查看器 `http://localhost:8080/viewer/`
2. 切换到"小红书"标签页
3. 观察笔记卡片的图片 src 属性
4. 对于有本地图片的笔记，检查是否使用 `/local-images/` 路径

**预期结果:**
- ✅ 有本地图片的笔记使用 `/local-images/xhs/...` 路径
- ✅ 图片正常显示

---

### TC-03: 本地图片不存在时自动 Fallback

**测试目的:** 验证本地图片加载失败时自动切换到远程 URL

**测试步骤:**
1. 打开浏览器开发者工具 (F12)
2. 切换到 Network 标签页
3. 访问 `http://localhost:8080/viewer/`
4. 观察图片请求：
   - 首先请求本地图片路径
   - 如果本地图片返回 404，应自动请求远程 URL

**预期结果:**
- ✅ 本地图片加载失败时触发 onerror 事件
- ✅ 自动切换到远程 URL 加载
- ✅ 不会出现无限循环请求

---

### TC-04: 图片加载状态动画

**测试目的:** 验证图片加载过程中的状态显示

**测试步骤:**
1. 打开 `http://localhost:8080/viewer/`
2. 清除浏览器缓存（模拟首次加载）
3. 刷新页面
4. 观察图片加载过程：
   - 加载中：显示骨架屏/闪烁动画
   - 加载完成：平滑过渡显示图片
   - 加载失败：显示错误占位符（🖼️）

**预期结果:**
- ✅ 图片加载中显示 shimmer 动画效果
- ✅ 图片加载成功后平滑过渡
- ✅ 图片加载失败显示占位符

---

### TC-05: 静态文件路由正确配置

**测试目的:** 验证 `/local-images/` 路由正确服务静态文件

**测试步骤:**
1. 直接访问本地图片路径，例如：
   `http://localhost:8080/local-images/xhs/images/xxx.jpg`
2. 检查是否正确返回图片文件

**预期结果:**
- ✅ 存在的图片文件正确返回
- ✅ 不存在的文件返回 404

---

### TC-06: 多平台图片显示

**测试目的:** 验证所有平台（小红书、抖音、B站、知乎）的图片显示

**测试步骤:**
1. 访问 `http://localhost:8080/viewer/`
2. 依次切换到各平台标签页
3. 观察每个平台的笔记卡片图片显示

**预期结果:**
- ✅ 小红书笔记图片正常显示
- ✅ 抖音视频封面正常显示
- ✅ B站内容封面正常显示
- ✅ 知乎内容图片正常显示
