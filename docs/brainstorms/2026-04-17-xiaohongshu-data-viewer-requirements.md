# 小红书数据展示前端网站

## Problem Frame

用户已经通过 MediaCrawler 爬取了小红书笔记数据（JSONL 格式），需要一个美观的前端网站来：
1. **展示爬取的数据** - 以瀑布流卡片形式浏览笔记
2. **监控爬取状态** - 实时查看爬取进度、统计信息

用户希望数据能动态更新——无论是爬取中还是爬取完成后，前端都能自动获取最新数据。

## Requirements

**数据展示**
- R1. 以瀑布流卡片布局展示笔记列表（类似小红书风格）
- R2. 每张卡片显示：封面图、标题、作者昵称、点赞数、收藏数
- R3. 点击卡片可查看笔记详情（完整正文、所有图片、标签）
- R4. 支持按关键词筛选笔记
- R5. 支持搜索笔记标题

**实时监控**
- R6. 显示爬取状态（运行中/已停止）
- R7. 显示统计信息：总笔记数、总图片数、各关键词笔记数
- R8. 显示最近爬取的笔记（实时更新）

**动态更新**
- R9. 使用轮询机制自动刷新数据（可配置间隔）
- R10. 支持手动刷新按钮
- R11. 显示最后更新时间

**用户体验**
- R12. 响应式设计，支持移动端浏览
- R13. 图片懒加载，优化性能
- R14. 美观的 UI，类似小红书风格

## Success Criteria

- 能够正确读取并展示 JSONL 数据文件
- 瀑布流布局在不同屏幕宽度下正常工作
- 轮询刷新能自动获取新增数据
- 统计信息准确反映数据状态
- 页面加载流畅，无明显卡顿

## Scope Boundaries

- 仅展示本地数据，不提供编辑功能
- 不实现用户认证
- 不提供数据导出功能（数据已存储在本地）
- 不支持视频播放（仅展示视频封面）

## Key Decisions

### 技术栈：纯 HTML + CSS + JavaScript

**理由：**
- 简单快速，无需构建工具和 npm 依赖
- 直接通过文件协议或简单 HTTP 服务器即可运行
- 维护成本低，无框架升级烦恼

### 数据更新：轮询刷新

**理由：**
- 实现简单，兼容性好
- 爬取频率较低（2-5秒/条），轮询足够实时
- 无需 WebSocket 复杂性

### 布局：瀑布流卡片

**理由：**
- 类似小红书原生体验，用户熟悉
- 适合展示图片为主的笔记内容
- CSS Grid 或 Masonry 布局可实现

### 后端：Python Flask 简单 API

**理由：**
- 项目已使用 Python，无额外语言学习成本
- Flask 轻量，适合简单 API 服务
- 提供数据读取、统计、状态查询接口

## Dependencies / Assumptions

### 依赖
- Python 3.8+（已有）
- Flask（需安装）
- 现有 JSONL 数据文件路径约定

### 假设
- 数据文件存储在 `MediaCrawler/data/xhs/jsonl/` 目录
- 图片已下载到 `MediaCrawler/data/xhs/images/` 目录
- 前端通过 HTTP API 访问数据（非直接读取文件）
