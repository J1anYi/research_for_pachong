## v2.0 Requirements (Completed)

### 任务管理

- [x] **TASK-01**: 创建图片下载任务数据库表（SQLite）✅ Phase 4
- [x] **TASK-02**: 任务状态管理 - 支持 pending, downloading, completed, failed 四种状态 ✅ Phase 4
- [x] **TASK-03**: 任务重试机制 - 最大重试次数配置,指数退避策略 ✅ Phase 4
- [x] **TASK-04**: 任务优先级支持 - 高优先级任务优先处理 ✅ Phase 4

### 消息队列

- [x] **QUEUE-01**: 实现图片下载消息队列（asyncio.Queue）✅ Phase 4
- [x] **QUEUE-02**: 消息消费者 Listener - 从队列取出消息,执行下载 ✅ Phase 4
- [x] **QUEUE-03**: 抖动间隔下载配置 - 可配置最小/最大间隔,避免限流 ✅ Phase 4
- [x] **QUEUE-04**: 并发下载控制 - 可配置最大并发数 ✅ Phase 4

### 定时任务

- [x] **SCHED-01**: 定时扫描未完成任务（asyncio 后台任务）✅ Phase 5
- [x] **SCHED-02**: 失败任务重试调度 - 超过退避时间后重新加入队列 ✅ Phase 5
- [x] **SCHED-03**: 任务超时检测 - 下载超时自动标记为失败 ✅ Phase 5

### 图片存储

- [x] **IMG-01**: 图片本地存储路径规划 - 按平台/日期组织目录结构 ✅ Phase 6
- [x] **IMG-02**: 图片下载去重 - 同一 URL 不重复下载,使用 URL hash 判断 ✅ Phase 6
- [x] **IMG-03**: 图片格式验证 - 下载后验证是否为有效图片 ✅ Phase 6
- [x] **IMG-04**: 存储空间管理 - 可配置最大存储空间,LRU 清理 ✅ Phase 6

### 前端集成

- [x] **FE-01**: API 返回本地图片路径 - notes API 返回 local_image_url 字段 ✅ Phase 7
- [x] **FE-02**: 前端优先使用本地图片 - 检查 local_image_url 是否存在 ✅ Phase 7
- [x] **FE-03**: 本地图片不存在时 fallback 到远程 URL - 保证图片始终能显示 ✅ Phase 7
- [x] **FE-04**: 图片加载状态显示 - 加载中/加载失败状态提示 ✅ Phase 7

### 爬虫集成

- [x] **CRAWL-01**: 爬虫发布图片下载任务消息 - 爬取完成后发送图片 URL 到队列 ✅ Phase 8 (小红书)
- [x] **CRAWL-02**: 支持多平台图片 URL 提取 - 小红书完成 ✅ Phase 8, 其他平台待扩展

## Out of Scope

| Feature | Reason |
|---------|--------|
| 视频下载 | 本次只处理图片,视频下载更复杂 |
| 分布式任务队列 | 单进程 asyncio.Queue 足够,不需要 Celery/RabbitMQ |
| 实时进度推送 | 后续版本考虑 |
