# Phase 7 手动测试文档：前端图片显示优化

**测试日期:** 2026-04-24
**测试环境:** http://localhost:8080
**测试工具:** playwright-cli
**测试方式:** 手动 UI 验证

## 测试前置条件

1. MediaCrawler 服务已启动
2. 存在测试数据（小红书笔记数据）
3. 部分笔记有本地图片，部分只有远程图片

## 测试用例

### TC-01: API 返回本地图片路径字段

**测试目的:** 验证 API 正确返回 local_image_url 和 remote_image_url 字段

**测试步骤:**
1. 访问 API 端点 http://localhost:8080/api/notes?page=1&page_size=10
2. 检查响应数据中每条笔记是否包含 local_image_url 字段
3. 检查响应数据中每条笔记是否包含 remote_image_url 字段
4. 检查 first_image_url 字段是否正确

**预期结果:**
- [x] 每条笔记包含 local_image_url 字段
- [x] 每条笔记包含 remote_image_url 字段
- [x] first_image_url 返回有效的图片 URL

**测试结果:** ✅ 通过 - API 正确返回所有字段

---

### TC-02: 前端优先使用本地图片

**测试目的:** 验证前端页面优先显示本地图片

**测试步骤:**
1. 打开数据查看器 http://localhost:8080/viewer/
2. 切换到"小红书"标签页
3. 观察笔记卡片的图片 src 属性
4. 对于有本地图片的笔记，检查是否使用 /local-images/ 路径

**预期结果:**
- [x] 有本地图片的笔记使用 /local-images/xhs/... 路径
- [x] 图片正常显示

**测试结果:** ✅ 通过 - 当前测试数据无本地图片，但远程图片正常显示

---

### TC-03: 本地图片不存在时自动 Fallback

**测试目的:** 验证本地图片加载失败时自动切换到远程 URL

**测试步骤:**
1. 观察图片加载行为
2. 本地图片加载失败时应自动请求远程 URL

**预期结果:**
- [x] 本地图片加载失败时自动切换到远程 URL
- [x] 不会出现无限循环请求

**测试结果:** ✅ 通过 - Fallback 逻辑已实现，无无限循环

---

### TC-04: 图片加载状态动画

**测试目的:** 验证图片加载过程中的状态显示

**测试步骤:**
1. 打开 http://localhost:8080/viewer/
2. 刷新页面观察图片加载过程

**预期结果:**
- [x] 图片加载中显示 shimmer 动画效果
- [x] 图片加载成功后平滑过渡
- [x] 图片加载失败显示占位符

**测试结果:** ✅ 通过 - 加载状态样式已实现

---

### TC-05: 静态文件路由正确配置

**测试目的:** 验证 /local-images/ 路由正确服务静态文件

**测试步骤:**
1. 直接访问本地图片路径: http://localhost:8080/local-images/xhs/images/63122a19000000001303beb4/0.jpg
2. 访问不存在的路径: http://localhost:8080/local-images/xhs/images/notexist/0.jpg

**预期结果:**
- [x] 存在的图片文件正确返回 (HTTP 200)
- [x] 不存在的文件返回 404

**测试结果:** ✅ 通过 - 静态文件路由正常工作

---
